# G-Vision OCR - Новая архитектура

## 🏗️ Архитектура: CNN Classifier + YOLO Detector

Переработана система распознавания текста с использованием **компонентного подхода**:

```
Исходное изображение
        ↓
┌─────────────────────────────────────┐
│ Препроцессинг (бинаризация)         │
└─────────────────────────────────────┘
        ↓
┌─────────────────────────────────────┐
│ Шаг 1: YOLO v8 ДЕТЕКТОР             │ ← Переиспользуем best.pt
│ Находит регионы с символами         │
└─────────────────────────────────────┘
        ↓
┌─────────────────────────────────────┐
│ Шаг 2: CNN КЛАССИФИКАТОР            │
│ EfficientNet-B0 для каждого региона │ ← Новое! Обучается на датасете
└─────────────────────────────────────┘
        ↓
┌─────────────────────────────────────┐
│ Шаг 3: Постобработка                │
│ - Коррекция по словарю              │
│ - Группировка в слова и строки      │
└─────────────────────────────────────┘
        ↓
Распознанный текст
```

---

## 📊 Преимущества нового подхода

| Параметр | Старый (YOLO классификация) | Новый (CNN) |
|----------|---------------------------|-----------|
| **Модель детекции** | YOLOv8l (547MB) | YOLOv8l (547MB) |
| **Модель классификации** | Встроена в YOLO (много параметров) | EfficientNet-B0 (~4MB) |
| **Время обучения** | ⏱️ 5-10 часов | ⏱️ 1-2 часа |
| **Потребление памяти** | 🔴 Много | 🟢 Мало |
| **Точность на русском** | ✅ Хорошо | ✅✅ Лучше (специализированна) |
| **Масштабируемость** | ⚠️ Смешаны задачи | ✅ Модульна |

## 🚀 Использование

### 1. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 2. Обучение классификатора

```bash
python train.py
```

**Этапы:**

- **Шаг 0**: Проверка окружения, загрузка словаря
- **Шаг 1**: Сканирование датасета (буквы, цифры, знаки)
- **Шаг 2**: Подготовка датасета для CNN (65×65 images)
- **Шаг 3**: Обучение EfficientNet-B0 (с早stopping)
- **Шаг 4**: Экспорт конфига
- **Шаг 5**: Тестирование на реальных фото

### 3. Файлы после обучения

```
g-vision-classifier.pt      ← Обученная CNN модель (~4MB)
g-vision-config.json         ← Конфиг (маппинг класов)
best.pt                      ← Детектор YOLO (переиспользуется)
```

## 🔧 Конфигурация

[train.py](train.py) класс `Config`:

```python
# CNN параметры
IMAGE_SIZE       = 64         # размер входа для CNN
CNN_EPOCHS       = 100        # эпохи обучения
CNN_BATCH_SIZE   = 32         # могно подогнать под вашу память
CNN_PATIENCE     = 15         # ранняя остановка

# Оптимизатор
LEARN_RATE       = 0.001      # начальная скорость обучения
WEIGHT_DECAY     = 0.00005    # регуляризация
```

## 📈 Метрики обучения

Во время обучения выводятся:

```
epoch   1  train_acc: 0.8234  val_acc: 0.7891  lr: 0.001000
epoch   2  train_acc: 0.8934  val_acc: 0.8456  lr: 0.000950
...
```

- `train_acc`: точность на тренировочном наборе
- `val_acc`: точность на валидационном наборе
- `lr`: текущий learning rate (понижается по косинусу)

## ⚙️ Модели

### YOLO Детектор
- **Модель**: YOLOv8l (текущие best.pt)
- **Задача**: Найти координаты каждого символа
- **Входные данные**: Подготовленное изображение (бинаризированное)
- **Выход**: Коробки около символов

### CNN Классификатор
- **Модель**: EfficientNet-B0 (из timm)
- **Задача**: Определить класс символа
- **Входные данные**: Изображение 64×64 (RGB)
- **Выход**: Вероятности для каждого класса

### Словарь
- **Файл**: russian_words.txt
- **Используется**: Для коррекции распознанных слов
- **Алгоритм**: Дамерау-Левенштейн расстояние

## 🐛 Решение проблем

### Проблема: Недостаточно памяти на GPU

**Решение:**
```python
C.CNN_BATCH_SIZE = 16  # Уменьшить batch size
C.IMAGE_SIZE = 48      # Уменьшить размер входа
```

### Проблема: Очень медленное обучение

**Решение:**
```python
C.CNN_BATCH_SIZE = 64  # Увеличить batch size
C.CNN_EPOCHS = 50      # Сначала мало эпох для тестирования
```

### Проблема: Overfitting (val_acc падает)

**Решение:**
- Увеличить `C.WEIGHT_DECAY` (больше регуляризации)
- Уменьшить `C.LEARN_RATE`
- Добавить больше аугментаций в DataLoader

## 📚 Дополнительно

### EfficientNet-B0
- Малая архитектура (4MB vs 547MB для YOLOv8l)
- Хороший баланс скорость/точность
- Хорошо работает на 64×64 изображениях

### CosineAnnealingLR
- Плавное снижение learning rate
- Помогает найти лучший минимум loss функции
- Автоматически реализован через scheduler

### Early Stopping
- Останавливает обучение если val_acc не улучшается
- Параметр: `C.CNN_PATIENCE = 15` эпох
- Сохраняет best_model автоматически
